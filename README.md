#智能家居 AI 服务助手

一个面向智能家居售后与用户服务场景的 AI 助手原型，支持知识库问答、工具调用与个性化使用报告生成。

## 项目简介

本项目聚焦扫地机器人使用过程中的常见问题与用户服务需求，设计并实现了一个基于 RAG（检索增强生成）和 Agent 工具调用的 AI 助手原型。

项目主要解决以下问题：

- 用户遇到故障时，难以快速找到对应解决方案
- 产品 FAQ、维护保养、选购指南等信息分散，查找成本较高
- 通用大模型直接回答容易产生幻觉，不适合直接用于服务场景
- 用户缺少个性化使用反馈与报告输出

为此，我搭建了一个支持知识库问答、外部数据查询和个性化报告生成的可交互 AI 服务助手 Demo，用于验证该场景下 AI 产品原型的可行性。

## 核心功能

### 1. 知识库问答
基于 FAQ、故障排除、维护保养、选购指南等文档构建向量知识库，支持用户围绕扫地机器人常见问题进行问答。

示例问题：
- 机器人迷路了怎么办？
- 扫地机器人建图错乱怎么办？
- 耗材多久需要更换？

### 2. 多工具调用
为 Agent 提供多个工具能力，包括：
- 用户位置获取
- 天气信息获取
- 用户 ID 获取
- 外部系统记录查询
- 报告生成上下文补充

使模型不仅能“回答”，还能根据任务自动调用外部能力完成更复杂的需求。

### 3. 个性化使用报告生成
结合用户 ID、月份和外部使用记录，支持自动生成用户使用报告，用于展示 AI 在用户运营、售后服务和个性化推荐场景中的应用潜力。

### 4. 流式输出体验
支持流式生成回答，优化用户等待过程中的交互体验，使原型更接近真实产品形态。

## 产品思路

本项目并不只是一个简单问答机器人，而是一个面向真实服务场景的 AI 产品原型。

整体思路如下：

用户输入问题  
→ Agent 判断任务类型  
→ 调用知识库检索 / 外部工具 / 报告生成流程  
→ 大模型组织最终回答  
→ 将结果以流式方式返回给用户

通过这种方式，将“知识获取”“外部信息调用”“个性化生成”整合到统一入口中，降低用户操作成本，提升服务效率。

## 技术方案

- **大模型**：Qwen / Chat Model
- **Agent 框架**：LangChain Agent
- **RAG 检索**：Chroma + Embedding Model
- **文本处理**：RecursiveCharacterTextSplitter
- **工具调用**：LangChain Tools
- **界面原型**：Streamlit
- **配置管理**：YAML 配置文件
